# hotyogapower.com

The public **storefront / front door** for HYP Yoga — its own static site, deliberately
separate from the `hyp.yoga` web app and backend.

> **Hard rule:** Anything built on `hyp.yoga` stays unchanged. This site is 100% additive.
> It is static, runs **zero JavaScript**, collects **no data**, and only *links* to hyp.yoga.

## Pages

| Path | File | Purpose |
|------|------|---------|
| `/` | `index.html` | Front door — brand hero + two doors (the app · the web) |
| `/app` | `app.html` | The iOS app — overview + "coming soon to the App Store" + tier ladder |
| `/web` | `web.html` | Gateway — Sign up → `hyp.yoga/signup`, Log in → `hyp.yoga/login` |

Design language is inherited from hyp.yoga (cream + warm gold, Cormorant Garamond + Inter,
self-hosted for GDPR). Shared styles live in `style.css`; fonts in `fonts/` (Cormorant + Inter
subset only). The OG image is a placeholder generated by `scripts/make-og.py` (Georgia stand-in
for Cormorant) — replace `images/og-image.png` with a designed asset when ready.

## Local preview

```bash
python3 scripts/preview-server.py        # serves on http://localhost:8765 with clean URLs
```
(A plain `python3 -m http.server` also works, but `/app` and `/web` clean URLs need the
preview server or a Netlify deploy.)

## Deploy (one-time setup)

This repo is a normal clone — **no** `index.lock` / `/tmp/hyp-deploy` workaround (that's
hyp-yoga-only). Just push; Netlify auto-deploys.

1. **GitHub repo**
   ```bash
   gh repo create joshuajudd-usa/hotyogapower --public \
     --source=/Users/joshua/Code/hotyogapower --remote=origin --push
   ```
   (or create it on github.com and `git remote add origin … && git push -u origin main`)

2. **Netlify site** — New site → *Import from Git* → pick `hotyogapower`.
   Build command: *(none)* · Publish directory: `.` · Functions: *(none)*.

3. **Domain** — Netlify → Domain settings → add `hotyogapower.com` **and** `www.hotyogapower.com`.
   Set the **apex (`hotyogapower.com`) as Primary** so `www` auto-301s to it.
   Point DNS at the registrar (Netlify nameservers, or the A/ALIAS + CNAME records Netlify shows).
   Let Netlify provision the Let's Encrypt cert.

4. **Verify** — visit `https://hotyogapower.com`; confirm the URL sticks (no redirect to hyp.yoga),
   both doors work, and `/web` lands on hyp.yoga signup/login.

Thereafter: edit → `git commit` → `git push` → Netlify rebuilds.

## Out of scope (would touch hyp.yoga or iOS — reauthorize separately)

- A launch waitlist / any data capture (would require a privacy disclosure on this domain).
- Phase 2: re-authored app info / tutorial / walkthrough / sales pages, built from hyp.yoga's
  informational pages as source material.
